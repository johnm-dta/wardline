## Task 3: Model Extensions + Annotation Fingerprint Computation

**Files:**
- Modify: `src/wardline/manifest/models.py` (add `artefact_class`, `last_changed` to `FingerprintEntry`; add `CoverageReport` dataclass)
- Modify: `src/wardline/manifest/schemas/fingerprint.schema.json` (add new fields, update envelope structure)
- Modify: `src/wardline/scanner/fingerprint.py` (add `compute_annotation_fingerprint`, `compute_single_annotation_fingerprint`, `batch_compute_fingerprints`)
- Create: `tests/unit/scanner/test_annotation_fingerprint.py`

**Model additions in `manifest/models.py`:**
```python
@dataclass(frozen=True)
class FingerprintEntry:
    qualified_name: str
    module: str
    decorators: tuple[str, ...]
    annotation_hash: str
    tier_context: int
    boundary_transition: str | None = None
    last_changed: str | None = None      # NEW — spec §9.2 temporal metadata
    artefact_class: str = ""             # NEW — "policy" or "enforcement" per §9.3.1

@dataclass(frozen=True)
class CoverageReport:
    """Annotation coverage statistics."""
    annotated: int
    total: int
    ratio: float
    tier1_annotated: int
    tier1_total: int
    tier1_unannotated: tuple[str, ...] = ()
```

**`CoverageReport` goes in `manifest/models.py`** (not `scanner/fingerprint.py`) to maintain import direction: scanner imports manifest.models.

**Three fingerprint functions in `scanner/fingerprint.py`:**

1. `compute_annotation_fingerprint(qualname, decorator_names, decorator_attrs) -> str` — 16-char hex hash.
   - **WARNING: Do NOT include file_path in payload.** This is a deliberate divergence from the existing `compute_ast_fingerprint` in the same file. The annotation fingerprint is path-agnostic (a function moved between files with unchanged annotations produces the same hash). Include `sys.version_info.major.minor` in the payload (same as AST fingerprint).
   - **Note:** `decorator_attrs` will be empty dicts for now — `WardlineAnnotation.attrs` is always `MappingProxyType({})` from static AST analysis. The hash spec includes attrs for forward compatibility with runtime introspection. The hash is still correct and deterministic with empty attrs.

2. `compute_single_annotation_fingerprint(file_path, qualname, manifest) -> FingerprintEntry | None` — single-function path for `explain` command. Discovers annotations for one file, computes hash for one function.

3. `batch_compute_fingerprints(root, manifest) -> tuple[list[FingerprintEntry], CoverageReport]` — walks .py files, calls `discover_annotations()` (requires parsing each file to AST first), computes hashes, counts coverage.

**Schema update:** Update `fingerprint.schema.json` to match the new baseline envelope format (design spec §2). Add `artefact_class` and `last_changed` to the entry schema. Run `jsonschema.validate()` on baseline load.

**Test cases:**
- `test_annotation_fingerprint_deterministic` — same inputs → same hash
- `test_annotation_fingerprint_changes_on_decorator_change` — different decorators → different hash
- `test_annotation_fingerprint_invariant_to_file_move` — same qualname+decorators, different module → same hash
- `test_annotation_fingerprint_sorted_decorators` — decorator order doesn't affect hash
- `test_annotation_fingerprint_includes_python_version` — version in payload
- `test_batch_compute_produces_entries` — returns list of FingerprintEntry
- `test_batch_compute_includes_coverage` — CoverageReport has correct counts
- `test_artefact_class_policy_for_tier_decorators` — boundary/tier → "policy"
- `test_artefact_class_enforcement_for_supplementary` — other → "enforcement"
- `test_single_function_fingerprint` — returns entry for specific qualname

**Definition of Done:**
- [ ] Annotation fingerprint matches hashing scheme spec algorithm (no file_path!)
- [ ] Single-function and batch APIs both work
- [ ] `CoverageReport` in `manifest/models.py`

**Metric functions (pure, no CLI concerns):**

- `collect_fingerprint_metrics(manifest_dir) -> FingerprintMetrics` — baseline presence, age, coverage (from stored `coverage` section in baseline JSON)
- `collect_manifest_metrics(manifest_path) -> ManifestMetrics` — version, ratification age/overdue, analysis level
- `collect_rule_metrics(manifest_path, config_path) -> RuleMetrics` — active rule count, disabled rules list, any disabled UNCONDITIONAL rules

**`governance_profile`:** Hardcode `"lite"` for now — the manifest model doesn't have this field yet. Add a `# TODO: read from manifest when field exists` comment.

**Read baseline files once:** All functions that read baselines load content once and pass as data, not as paths that get re-read (avoids TOCTOU window flagged by Security).
