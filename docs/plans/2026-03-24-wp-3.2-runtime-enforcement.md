# WP 3.2: Runtime Enforcement Hooks — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Opt-in runtime tier enforcement — automatic decorator stamping, TierStamped[T] frozen wrapper, checking functions, WardlineBase integration.

**Architecture:** Three-layer design: TAINT_TO_TIER bridge (core) → decorator auto-stamping + explicit API (runtime/enforcement) → checking functions + WardlineBase hooks (runtime). All behind opt-in flag.

**Tech Stack:** Python 3.12+, existing wardline decorator/runtime infrastructure

**Prerequisites:**
- WP 2.2 (NewType migration) shipped
- Design spec approved: `docs/superpowers/specs/2026-03-24-runtime-enforcement-design.md`

**Key design decisions (from 3 rounds of panel review, 20 individual reviewer dispatches):**
- `TierStamped[T]` frozen generic wrapper is the SOLE tier-aware container (5-2 panel vote over dict subclasses)
- `TierViolationError` inherits `Exception`, NOT `TypeError` (security: broad `except TypeError` can't swallow violations)
- `_try_stamp_tier` catches both `AttributeError` and `TypeError` (AST `__slots__`, frozen dataclasses)
- `stamp_tier()` has `overwrite=False` default (prevents silent re-stamp escalation)
- `disable()` has call-once latch (can't turn off enforcement after first check)
- Auto-wrapping: decorator returns `TierStamped(result)` for unstampable types (dicts, primitives)
- `_wardline_groups` normalized to `tuple(sorted(...))` at stamp time
- WARNING log on non-None unstampable skip + audit log before every raise

## Task 2: Rewrite `runtime/enforcement.py` — Core Infrastructure

**Files:**
- Rewrite: `src/wardline/runtime/enforcement.py`
- Rewrite: `tests/unit/runtime/test_enforcement.py`

This is a FULL REWRITE of the broken first implementation.

**Module structure:**

```
# Enable/disable with call-once latch
_enforcement_enabled: bool
_first_check_done: bool = False
enable(), disable(), is_enabled()

# TierViolationError(Exception) — NOT TypeError
class TierViolationError(Exception): ...

# TierStamped[T] — frozen generic wrapper
@dataclass(frozen=True, slots=True)
class TierStamped(Generic[T]): ...

# Stamping functions
stamp_tier(obj, tier, *, groups=(), stamped_by="", overwrite=False)
unstamp(obj) -> Any

# Checking functions
check_tier_boundary(obj, *, expected_min_tier, context="")
check_validated_record(obj)

# Optional violation callback
on_violation: Callable | None = None

# WardlineBase construction enforcement
enforce_construction(instance)
check_subclass_tier_consistency(cls) -> list[str]
```

**Critical implementation details:**

1. **`TierViolationError(Exception)`** — NOT TypeError. Includes `obj`, `expected_tier`, `actual_tier` attributes.

2. **`TierStamped[T]`** — `frozen=True, slots=True`. `__post_init__` validates tier 1-4. Generic type parameter for mypy.

3. **`stamp_tier()`** — `overwrite=False` default. If `_wardline_tier` exists and `overwrite=False`, raises `ValueError`. Normalizes groups to `tuple(sorted(groups))`.

4. **`check_tier_boundary()`** — Logs WARNING before raising. Calls `on_violation` if set. No-op when disabled. Sets `_first_check_done = True` on first call.

5. **`check_validated_record()`** — Post-isinstance validation: `_wardline_tier` must be int 1-4, `_wardline_groups` must be tuple or set of ints. Logs before raising.

6. **`disable()`** — If `_first_check_done` is True, raises `RuntimeError("Enforcement state cannot be changed after first use")`. Idempotent before first check.

7. **`enforce_construction(instance)`** — Reads `_wardline_tier_source` (NOT `_wardline_transition_from_tier`!) from decorated methods. Maps via `TAINT_TO_TIER`. Reads `_wardline_transition` for bridging. Logs WARNING on mixed tiers.

## Task 4: Decorator Auto-Stamping (`decorators/_base.py`)

**Files:**
- Modify: `src/wardline/decorators/_base.py`
- Create: `tests/unit/decorators/test_auto_stamping.py`

**Changes to `wardline_decorator` factory:**

1. At decorator construction time (outer scope): compute `output_tier` from `_wardline_transition[1]` or `_wardline_tier_source` via `TAINT_TO_TIER`. Store as closure variable. If no tier derivable (supplementary decorator), `output_tier = None`.

2. In wrapper function, after `result = unwrapped(...)`:
```python
if output_tier is not None:
    from wardline.runtime.enforcement import is_enabled
    if is_enabled() and result is not None:
        result = _try_stamp_tier(result, output_tier, output_groups, stamped_by)
```

3. `_try_stamp_tier` implementation:
- Try `setattr(result, "_wardline_tier", output_tier)` etc.
- On `AttributeError` or `TypeError`: log WARNING, return `TierStamped(value=result, _wardline_tier=output_tier, ...)` instead
- Normalize groups: `tuple(sorted(getattr(wrapper, "_wardline_groups", set())))`
- `stamped_by = wrapper.__qualname__`

**Key: `_try_stamp_tier` returns the (possibly wrapped) result.** The decorator wrapper uses the return value: `return _try_stamp_tier(result, ...)` when enforcement is on.
