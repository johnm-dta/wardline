# WP 3.2: Runtime Enforcement Hooks ŌĆö Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Opt-in runtime tier enforcement ŌĆö automatic decorator stamping, TierStamped[T] frozen wrapper, checking functions, WardlineBase integration.

**Architecture:** Three-layer design: TAINT_TO_TIER bridge (core) ŌåÆ decorator auto-stamping + explicit API (runtime/enforcement) ŌåÆ checking functions + WardlineBase hooks (runtime). All behind opt-in flag.

**Tech Stack:** Python 3.12+, existing wardline decorator/runtime infrastructure

**Prerequisites:**
- WP 2.2 (NewType migration) shipped
- Design spec approved: `docs/superpowers/specs/2026-03-24-runtime-enforcement-design.md`

**Fourth-pass panel review (7 reviewers on implementation plan) ŌĆö fixes incorporated:**
- **Python C-1:** `output_groups` must be read at CALL TIME from `wrapper._wardline_groups`, NOT captured at construction time
- **Python C-2 + Systems C-1:** Auto-wrapping changes return types ŌĆö callers must use `unstamp()`. Integration test required.
- **Security C-1 + Python I-5:** `_first_check_done` must be set by ALL three check functions, not just `check_tier_boundary`
- **SA I-3:** Task 4 depends on Task 2 (needs TierStamped class) ŌĆö dependency graph fixed
- **SA I-1 + Static Analysis C-1:** Third DECORATOR_TAINT_MAP mismatch: `authoritative_construction` (PIPELINEŌåÆAUDIT_TRAIL)
- **QE C-1:** Spec/plan contradiction on skip-vs-wrap resolved: auto-wrap is correct behavior
- **IRAP C-2 + QE S-1:** Test fixture teardown needs `_reset_enforcement_state()` for latch compat
- **SA S-3 + QE I-3:** `test_tier_stamped_nested_blocked` added to Task 2
- **Security I-2:** `stamp_tier()` must log before raising TypeError on frozen objects

**Key design decisions (from 3 rounds of panel review, 20+ individual reviewer dispatches):**
- `TierStamped[T]` frozen generic wrapper is the SOLE tier-aware container (5-2 panel vote over dict subclasses)
- `TierViolationError` inherits `Exception`, NOT `TypeError` (security: broad `except TypeError` can't swallow violations)
- `_try_stamp_tier` catches both `AttributeError` and `TypeError` (AST `__slots__`, frozen dataclasses)
- `stamp_tier()` has `overwrite=False` default (prevents silent re-stamp escalation)
- `disable()` has call-once latch (can't turn off enforcement after first check)
- Auto-wrapping: decorator returns `TierStamped(result)` for unstampable types (dicts, primitives)
- `_wardline_groups` normalized to `tuple(sorted(...))` at stamp time
- WARNING log on non-None unstampable skip + audit log before every raise

## Task 2: Rewrite `runtime/enforcement.py` ŌĆö Core Infrastructure

**Files:**
- Rewrite: `src/wardline/runtime/enforcement.py`
- Rewrite: `tests/unit/runtime/test_enforcement.py`

This is a FULL REWRITE of the broken first implementation.

**Module structure:**

```
# Enable/disable with call-once latch
_enforcement_enabled: bool
_first_check_done: bool = False
enable(), disable(), is_enabled()

# TierViolationError(Exception) ŌĆö NOT TypeError
class TierViolationError(Exception): ...

# TierStamped[T] ŌĆö frozen generic wrapper
@dataclass(frozen=True, slots=True)
class TierStamped(Generic[T]): ...

# Stamping functions
stamp_tier(obj, tier, *, groups=(), stamped_by="", overwrite=False)
unstamp(obj) -> Any

# Checking functions
check_tier_boundary(obj, *, expected_min_tier, context="")
check_validated_record(obj)

# Optional violation callback
on_violation: Callable | None = None

# WardlineBase construction enforcement
enforce_construction(instance)
check_subclass_tier_consistency(cls) -> list[str]
```

**Critical implementation details:**

1. **`TierViolationError(Exception)`** ŌĆö NOT TypeError. Includes `obj`, `expected_tier`, `actual_tier` attributes.

2. **`TierStamped[T]`** ŌĆö `frozen=True, slots=True`. `__post_init__` validates tier 1-4. Generic type parameter for mypy.

3. **`stamp_tier()`** ŌĆö `overwrite=False` default. If `_wardline_tier` exists and `overwrite=False`, raises `ValueError`. Normalizes groups to `tuple(sorted(groups))`.

4. **`check_tier_boundary()`** ŌĆö Logs WARNING before raising. Calls `on_violation` if set. No-op when disabled. Sets `_first_check_done = True` on first call. ALL three checking entry points (`check_tier_boundary`, `check_validated_record`, `enforce_construction`) must set `_first_check_done`.

4a. **`stamp_tier()`** ŌĆö Must also log WARNING before raising TypeError on frozen objects (audit trail requirement from security review). The raw TypeError must not propagate unlogged.

4b. **`_reset_enforcement_state()`** ŌĆö Internal function (not exported) that resets `_enforcement_enabled` and `_first_check_done` to defaults. Used ONLY by test fixtures. Not part of public API. Without this, the call-once latch makes test teardown impossible.

5. **`check_validated_record()`** ŌĆö Post-isinstance validation: `_wardline_tier` must be int 1-4, `_wardline_groups` must be tuple or set of ints. Logs before raising.

6. **`disable()`** ŌĆö If `_first_check_done` is True, raises `RuntimeError("Enforcement state cannot be changed after first use")`. Idempotent before first check.

7. **`enforce_construction(instance)`** ŌĆö Reads `_wardline_tier_source` (NOT `_wardline_transition_from_tier`!) from decorated methods. Maps via `TAINT_TO_TIER`. Reads `_wardline_transition` for bridging. Logs WARNING on mixed tiers.

## Task 4: Decorator Auto-Stamping (`decorators/_base.py`)

**Files:**
- Modify: `src/wardline/decorators/_base.py`
- Create: `tests/unit/decorators/test_auto_stamping.py`

**Changes to `wardline_decorator` factory:**

1. At decorator construction time (outer scope): compute `output_tier` from `_wardline_transition[1]` or `_wardline_tier_source` via `TAINT_TO_TIER`. Store as closure variable. If no tier derivable (supplementary decorator), `output_tier = None`.

2. In wrapper function, after `result = unwrapped(...)`:
```python
if output_tier is not None:
    from wardline.runtime.enforcement import is_enabled
    if is_enabled() and result is not None:
        # output_groups read at CALL TIME (not construction time!)
        # because stacked decorators accumulate groups across the chain
        current_groups = tuple(sorted(getattr(wrapper, "_wardline_groups", set())))
        stamped_by = wrapper.__qualname__  # = original function's qualname via functools.wraps
        result = _try_stamp_tier(result, output_tier, current_groups, stamped_by)
```

**CRITICAL: `output_groups` is read from `wrapper._wardline_groups` inside the wrapper body at call time, NOT captured as a closure alongside `output_tier`.** This is because `_wardline_groups` accumulates across stacked decorators (copy-on-add in the factory), so the construction-time value would miss outer decorator groups.

**CRITICAL: `output_tier` IS captured at construction time** ŌĆö it comes from `semantic_attrs` (the kwargs to `wardline_decorator`), NOT from `getattr(fn, ...)`. Each decorator independently knows its own tier from its own `_wardline_transition` or `_wardline_tier_source`.

3. `_try_stamp_tier` implementation:
- Try `setattr(result, "_wardline_tier", output_tier)` etc.
- On `AttributeError` or `TypeError`: log WARNING, return `TierStamped(value=result, _wardline_tier=output_tier, ...)` instead
- If result already has `_wardline_tier` (pre-stamped by callee decorator), do NOT overwrite ŌĆö respect the callee's stamp (innermost tier wins). `_try_stamp_tier` must catch `ValueError` from the `overwrite=False` default on `stamp_tier()` and silently return the pre-stamped result. This prevents stacked-decorator escalation while avoiding exceptions propagating to callers.

**Key: `_try_stamp_tier` returns the (possibly wrapped) result.** The decorator wrapper uses the return value: `return _try_stamp_tier(result, ...)` when enforcement is on.

**CALLER BREAKAGE WARNING:** When enforcement is enabled, decorated functions returning dicts/primitives/frozen objects will return `TierStamped[T]` instead of `T`. Callers must use `unstamp()` before consuming. This is a deliberate design choice ŌĆö the unwrap IS the trust boundary crossing. Document in decorator docstrings and migration guide.
