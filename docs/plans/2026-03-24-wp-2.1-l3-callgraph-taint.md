# WP 2.1: L3 Call-Graph Taint — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add intra-module call-graph taint propagation (Level 3) — refining function-level taints by analyzing what functions call.

**Architecture:** Three-way provenance classification (decorator/module_default/fallback), total trust order via `least_trusted()`, SCC-based fixed-point propagation, mandatory taint provenance tracking.

**Tech Stack:** Python 3.12+, stdlib `ast` only (no `astroid`), Hypothesis for property-based tests

**Prerequisites:**
- v0.2.1 shipped (corpus verifier taint wiring, analysis_level_required)
- Design spec approved: `docs/superpowers/specs/2026-03-24-l3-callgraph-taint-design.md`

**Key design decisions (from two rounds of 7-reviewer panel):**
- Decorated functions are **anchored** (immutable seeds) — core security property
- Module_tiers functions are **floating-downward** — can be demoted but not upgraded
- Fallback (UNKNOWN_RAW) functions are **floating-unconstrained** — L3's primary value
- Unresolved calls are **transparent** (no taint contribution) with `L3-LOW-RESOLUTION` diagnostic
- Provenance is **mandatory** on ALL functions (assessment evidence, not optional)

**Third-pass panel review (7 reviewers on implementation plan) — fixes incorporated:**
- **Python C-1 + SA C-1:** L1 return-type change is a HARD BREAK, not transparent — all call sites must update atomically
- **Static Analysis C-1 + Security I-3:** Module_default functions must initialize to L1 taint (not UNKNOWN_RAW) and propagation must clamp result at L1 floor: `max(max_rank(callees), L1_rank)`
- **SA C-2/C-3 + Security C-1:** Spec function signatures are stale — plan signatures are authoritative, not spec stubs
- **Systems C-1/C-2:** Task 9 (migration) must precede Task 11 (baselines); `migrate` must stamp `analysis_level`
- **IRAP C-1/C-2:** Schema updates for `analysis_level` + `migrated_from`; `apply_exceptions` needs active level parameter
- **Security I-4 + IRAP I-3:** L3 failure and convergence-bound must emit TOOL-ERROR/ERROR findings (not just log)
- **QE C-1/C-2/C-3:** Missing tests added; Hypothesis monotonicity scoped to module_default only
- **SA I-2 + IRAP I-4:** Provenance records must be seeded for ALL functions (anchored + unrefined)

## Task 2: Trust Order + `least_trusted()`

**Files:**
- Create: `src/wardline/scanner/taint/callgraph.py`
- Create: `tests/unit/scanner/test_callgraph.py`

**What:** Define the total trust order and the `least_trusted(a, b)` function.

**Implementation:**
```python
TRUST_RANK: dict[TaintState, int] = {
    TaintState.AUDIT_TRAIL: 0,
    TaintState.PIPELINE: 1,
    TaintState.SHAPE_VALIDATED: 2,
    TaintState.UNKNOWN_SEM_VALIDATED: 3,
    TaintState.UNKNOWN_SHAPE_VALIDATED: 4,
    TaintState.EXTERNAL_RAW: 5,
    TaintState.UNKNOWN_RAW: 6,
    TaintState.MIXED_RAW: 7,
}

assert len(TRUST_RANK) == len(TaintState), "TRUST_RANK must cover all TaintState members"

def least_trusted(a: TaintState, b: TaintState) -> TaintState:
    """Return the less-trusted of two taint states (higher rank)."""
    return a if TRUST_RANK[a] >= TRUST_RANK[b] else b
```

**Tests:**

- `test_least_trusted_identity` — `least_trusted(a, a) == a` for all states
- `test_mixed_raw_is_bottom` — `least_trusted(MIXED_RAW, X) == MIXED_RAW` for all X

## Task 3: Call Graph Extraction

**Files:**
- Modify: `src/wardline/scanner/taint/callgraph.py` (add to file from Task 2)
- Modify: `tests/unit/scanner/test_callgraph.py` (add to file from Task 2)

**What:** `extract_call_edges(tree, qualname_map) -> tuple[dict[str, set[str]], dict[str, int], dict[str, int]]` — returns adjacency list, resolved call counts per function, and unresolved call counts per function.

**Qualname map:** Extract `ScanEngine._build_qualname_map()` and `_walk_with_scope()` to `scanner/_qualnames.py` as a shared utility. This is a required first step in Task 3, not optional. **The extraction must convert the recursive implementation to iterative** (the existing `_walk_with_scope` is recursive and may hit Python's recursion limit on large modules). Update `engine.py` to import from the shared utility. The function returns `dict[int, str]` keyed by `id(node)`.

**Return type expanded:** `extract_call_edges(tree, qualname_map) -> tuple[dict[str, set[str]], dict[str, int], dict[str, int]]` — adjacency list + resolved counts + unresolved counts per function. **Note:** The design spec's Section 4 shows a simpler return type (adjacency only) — this is stale. Use the plan's signature.

**Resolution rules:**
1. Build reverse maps: `{bare_name: qualname}` for module-level defs, `{class_name: {method_name: qualname}}` for methods
2. For each function body, walk `ast.Call` nodes:
   - `ast.Name(id=X)` where X in module defs → resolved edge
   - `ast.Attribute(value=ast.Name(id='self'), attr=M)` → look up class from caller qualname prefix → resolved edge if found
   - `ast.Name(id=ClassName)` where ClassName is a class → edge to `ClassName.__init__` if exists
   - Everything else → unresolved (increment count, no edge)

**Use iterative AST walk** (not recursive) to avoid recursion limit on large function bodies. Use `ast.walk()` which is iterative.

**Tests:**
- `test_direct_call_resolved` — `foo()` where `def foo` exists → edge
- `test_self_method_resolved` — `self.process()` in class method → edge to `ClassName.process`
- `test_constructor_resolved` — `MyClass()` → edge to `MyClass.__init__`
- `test_import_call_unresolved` — `json.loads()` → no edge, unresolved count incremented
- `test_parameter_call_unresolved` — `callback()` where callback is a parameter → no edge
- `test_nested_function_call` — inner function calling outer function → resolved
- `test_no_duplicate_edges` — same call twice → single edge (set semantics)
- `test_resolved_unresolved_counts` — correct counts returned

**Definition of Done:**

## Task 5: Fixed-Point Propagation

**Files:**
- Modify: `src/wardline/scanner/taint/callgraph_propagation.py` (add to file from Task 4)
- Modify: `tests/unit/scanner/test_callgraph_propagation.py` (add to file from Task 4)

**What:** `propagate_callgraph_taints(edges, taint_map, taint_sources, resolved_counts, unresolved_counts) -> tuple[dict[str, TaintState], dict[str, TaintProvenance]]`

**NOTE:** The design spec Section 5 shows a stale 2-argument signature. Use this plan's 5-argument signature — it is authoritative.

**Implementation:** Per spec Section 5 algorithm, with panel corrections:
1. Classify functions using `taint_sources` (decorator → anchored, module_default → floating-downward, fallback → floating-unconstrained)
2. **Initialize to L1 taints** — copy `taint_map` unchanged. Do NOT initialize floating functions to `UNKNOWN_RAW`. Module_default functions start at their L1 taint (this is their floor).
3. **Build reverse edge map** from forward adjacency list — needed for efficient worklist caller notification
4. SCC decomposition (Task 4)
5. Worklist iteration per SCC with safety bound
6. **Floor clamp for module_default:** After computing `new_taint = max_rank(callee_taints)`, clamp: `new_taint = max(new_taint_rank, L1_rank)` for module_default functions. This enforces "downward only."
7. Post-fixed-point assertions
8. **Seed provenance records for ALL functions** — anchored functions get `source="decorator"`, unrefined module_default get `source="module_default"`, unrefined fallback get `source="fallback"`. Only L3-refined functions get `source="callgraph"`. Provenance is mandatory on every function in the output.

**`via_callee` tie-breaking rule:** When multiple callees influence the result, `via_callee` records the callee with the highest trust rank (least trusted) — the one that dominates the `max_rank` computation. Deterministic: if tied, use alphabetically-first qualname.

**`TaintProvenance` dataclass** — define in `callgraph_propagation.py`:
```python
@dataclass(frozen=True)
class TaintProvenance:
    source: Literal["decorator", "module_default", "callgraph", "fallback"]
    via_callee: str | None = None
    resolved_call_count: int = 0
    unresolved_call_count: int = 0
```
